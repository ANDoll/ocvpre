# AEDL 对抗规避检测层：开发历程与算法设计报告

## 摘要

本文档记述 AEDL（Adversarial Evasion Detection Layer，对抗规避检测层）模块从需求分析到算法定型的完整研发过程。AEDL 部署于既有 NSFW Detector 视频审核系统上游，作为独立中间层，旨在识别并还原内容创作者常用的低成本规避手段（镜像翻转、边缘遮挡、画质降质、画面分屏/屏中屏等），将审核模型的有效覆盖范围从「理想样本」扩展到「对抗样本」。报告重点阐述了融合策略的迭代演进，以及在召回率与准确率权衡中基于"宁可误报也不放过"原则的最终方案选择。

---

## 一、开发历程概述

AEDL 的研发围绕一个核心矛盾展开：**下游审核模型对对抗样本（经规避处理的视频）的识别能力存在固有局限，而 AEDL 作为前置层无法替代模型本身**。整个开发历程可分为三个阶段。

### 阶段一：基础架构搭建

基于「方案一」设计文档，实现五段式处理管线：

- **模块 A（输入感知）**：毫秒级 CV 算子，检测镜像、边缘遮挡、画质降质、分屏等规避特征
- **模块 B（策略路由）**：决策表驱动的异常→变换映射，附加保险机制（默认对所有视频追加 mirror/vflip/brighten 送审）
- **模块 C（变换还原）**：对原始视频施加逆向变换，生成最多 8 个还原版本
- **后端送审**：原始+还原版本并行调用既有 `/api/v1/detect` 接口
- **模块 D（一致性校验）**：JSD 分布偏移、单标签跃升、规避分加权
- **模块 E（结果融合）**：保守最大值融合，保持原 API 响应契约

此阶段解决了前端零改动兼容、保险机制覆盖等问题，但在真实对抗样本测试中暴露了感知层与融合层的双重不足。

### 阶段二：感知层增强与屏中屏专项攻坚

**攻坚对象的选取依据**：在测试视频集（含半透明化、暗光蒙版、分屏、镜像翻转、闪烁插入帧、屏中屏等规避样本）的实测中发现，镜像翻转、调亮/调暗画面、加边框遮挡这一类常规人为作弊手段对原审核模型的影响有限——原模型基本仍能识别。但**屏中屏规避（将有害画面缩小后嵌入无害背景）对原模型造成了显著且严重的检测能力下降**，原模型对缩小画面的识别几乎失效（分数从安全区直接跌入低分区）。因此，AEDL 将屏中屏作为重点攻克问题，在感知、变换、融合三个层面均设计了专项机制。

针对该问题重点增强：

1. **感知层**：新增垂直翻转、暗光蒙版、闪烁插入帧检测；屏中屏检测采用块方差分析（12×12 网格 + 形态学开运算 + 连通域）配合轮廓检测备选，并引入帧差法作为动态窗口的兜底检测
2. **变换层**：新增 Unsharp Mask 锐化、CLAHE 对比度增强；屏中屏提取后放大到原始尺寸（INTER_LANCZOS4），让模型看到的画面尺寸与正常视频一致；设计 3 种分屏组合变换链
3. **一致性校验**：下调 `label_jump_threshold: 0.4→0.15`、`label_cliff_threshold: 0.5→0.3`、`restored_high_confidence: 0.8→0.5`，提升对还原版本信号的敏感度

### 阶段三：融合策略迭代与定案

屏中屏视频因原模型对缩小画面的检测能力有限，还原后 `anomaly_score` 仍普遍低于 0.5 阈值。融合策略经历了多次演进，最终基于"宁可误报也不放过"原则确定最终版方案。详见第四、五节。

## 二、需求分析

### 2.1 业务背景

视频审核系统面临日益精细的低成本规避手段。攻击者通过镜像翻转、画面分屏/屏中屏（将有害画面缩小嵌入无害背景）、暗光蒙版、闪烁插入帧等方式，使审核模型的识别率显著下降，而原始模型对这些对抗样本缺乏鲁棒性。

### 2.2 核心需求

1. **零数据增强**：不依赖标注数据，不引入模型训练，仅基于传统 CV 算子
2. **模型无关性**：作为独立预处理模块，与下游审核模型完全解耦，可复用
3. **前端零改动**：响应结构保持原 `{detection, report}` 顶层契约，规避分析作为可选 `aedl_analysis` 字段挂载
4. **保守召回**：任一还原版本揭示的违规信号均不应被遗漏
5. **可解释证据链**：输出具体规避类型与量化证据，支撑人工复核

### 2.3 关键约束

- AEDL 无法纠正原模型对类别的误判（如把暴力视频误标为辱骂/金钱）
- AEDL 无法让原模型识别它本来识别不到的内容
- 原模型在 `anomaly_score < 0.5`（is_harmful=false）时不触发 CLIP re-rank，`category_scores` 基于 SVLA logits2 的条件概率分配，在低分区间存在偏置

## 三、算法设计

### 3.1 五段式处理管线

```
[前端上传视频]
       │
       ▼
┌────────────────────────────┐
│  模块 A: 输入感知层          │  ← 快速 CV 算子，<50ms
│      ↓                    │
│  模块 B: 策略路由层          │  ← 决策表驱动 + 白名单 + 保险机制
│      ↓                    │
│  模块 C: 变换还原层          │  ← 生成最多 8 个还原版本
│      ↓                    │
│  调用原 /api/v1/detect     │  ← 原始+还原版本并行送审
│      ↓                    │
│  模块 D: 一致性校验层        │  ← JSD / 单标签跃升 / 规避分
│      ↓                    │
│  模块 E: 结果融合输出        │  ← 双融合策略 + 证据链
└────────────────────────────┘
       │
       ▼
[融合响应返回前端]
```

### 3.2 屏中屏检测与还原

屏中屏（picture-in-picture）是本项目的重点攻坚对象：攻击者将有害画面缩小后嵌入无害背景，原模型对缩小画面的检测能力显著下降。

**检测**：块方差分析法将灰度图划分为 12×12 块，计算每块方差；高方差区域（视频内容）被低方差区域（均匀背景）包围即判定为屏中屏。形态学开运算去除孤立噪声，连通组件分析定位窗口边界。配合帧差法（采样多帧累加相邻帧差异，静态背景无差异而动态窗口高差异）作为动态窗口的兜底检测。

**还原**：提取窗口后放大到原始尺寸（INTER_LANCZOS4 插值），让模型看到的画面尺寸与正常视频一致；叠加亮度增强、Unsharp Mask 锐化、CLAHE 对比度增强，生成 3 种组合变换链，从不同角度增强放大的有害画面。

### 3.3 融合策略演进

融合策略是 AEDL 设计的核心难点，经历了四个版本的迭代：

| 版本 | 策略 | 局限 |
|------|------|------|
| v1 | 保守最大值（仅 triggered_labels 融合） | 还原版本信号被严格条件过滤，低分区间无法突破阈值 |
| v2 | 双融合（category_scores 严格 + anomaly_score 宽松含 calibrated 兜底） | 仍依赖原模型真实分数，屏中屏视频普遍低于 0.5 |
| v3 | 证据累积（evidence_score = Σ restored × log2(ratio)，target = 0.25 + evidence） | 构造映射公式缺乏理论依据，本质为"凑结果" |
| v4（README 版） | 分屏强制机制（检测到 split_screen 即 anomaly=0.5） | 一刀切，不区分有无真实信号；对未检测到 split_screen 的样本无效 |
| **最终版** | **CLIP re-rank + 按涨幅精准放大** | 见第五节 |

## 四、关键决策：中间版本的局限

### 4.1 README 强制机制版（21:05-21:18 测试版）

该版本的核心机制为：**检测到 `split_screen` 异常时，强制将 `anomaly_score` 提升到 0.5（MEDIUM 等级），确保进入人工审核**。

测试结果：
- **暴力屏中屏视频（202606281804(9).mp4）**：屏中屏被成功检测 → 触发强制机制 → `anomaly_score=0.5` → 判为有害 ✓
- **金钱欺诈屏中屏视频（202606281804(12).mp4）**：屏中屏检测能力不足，仅检测到 flicker 异常，未触发 `split_screen` → 强制机制未生效 → `anomaly_score` 保持原模型低分输出 → **漏检** ✗

### 4.2 局限性分析

强制机制版存在两个根本问题：

1. **依赖感知层的检测能力**：若屏中屏未被检测到（窗口较小、边缘软化、背景有纹理），强制机制完全不生效，导致漏检
2. **一刀切不区分信号**：即使检测到屏中屏，强制提升到 0.5 也未基于还原版本的真实信号，缺乏可解释性

更深层的问题在于：**原模型对屏中屏内的有害内容识别能力本身不足**。即使 AEDL 正确还原了视频并让原模型重新检测，原模型输出的 `category_scores` 在低分区间仍存在 SVLA logits2 偏置——真实类别（如 Violent）涨幅小，误报类别（如 Money）涨幅反而最大。简单的"按涨幅放大"会放大误报而非真实信号。

## 五、最终版方案：CLIP re-rank + 按涨幅精准放大

### 5.1 设计动机

经分析原 NSFW Detector 模型推理代码（`pipeline/inference.py`），发现关键事实：

> 原模型在 `anomaly_score >= 0.5`（is_harmful=true）时触发 **CLIP re-rank**，用纯 CLIP 文本特征重新计算 `category_scores`，覆盖 SVLA logits2 的偏置分配；而当 `anomaly_score < 0.5` 时**不触发 re-rank**，`category_scores` 保留 logits2 偏置。

屏中屏视频的所有版本（原始+还原）`anomaly_score` 普遍低于 0.5，均未触发原模型 re-rank，因此 `category_scores` 不可靠。这是"按涨幅放大"会放大误报的根本原因。

### 5.2 方案设计

最终版采用两层机制叠加：

**第一层：AEDL 侧自做 CLIP re-rank**

- 复用网页模型的 CLIP ViT-B/16 与 7 类文本 prompts（每类 3 prompt 平均）
- 对屏中屏模式下所有 `anomaly_score < 0.5` 的版本，AEDL 自行抽帧 → CLIP 视觉特征 → 与文本特征计算相似度 → 重算 `category_scores`
- `category_scores[cat] = anomaly_score × max(0, topk_sim)`
- 用纯 CLIP 信号覆盖 SVLA logits2 偏置，获得更准确的类别分数

**第二层：按涨幅精准放大**

- 屏中屏模式下，对每个类别比较还原前后的分数，计算真实涨跌倍数
- 涨得越多 → 放大系数越大（3倍→×5，10倍→×10，50倍→×20，200倍→×25）
- 涨得少或不涨 → 不放大（保留真实分数）
- `final_score = min(max_restored × boost, 1.0)`

### 5.3 选择依据

基于**"宁可误报也不能放过"**原则，最终版相比中间版本的优势：

| 维度 | README 强制机制版 | 最终版 |
|------|------------------|--------|
| 暴力屏中屏 | 依赖感知层检测，能识别 | CLIP re-rank 修正类别 + 涨幅放大，能识别 |
| 金金钱欺诈屏中屏 | 感知层漏检则完全失效 | CLIP re-rank 不依赖单次检测，能识别 |
| 类别准确性 | 不区分信号，无类别 | CLIP re-rank 提供更准类别 |
| 可解释性 | 一刀切 0.5，无证据 | 基于真实涨幅放大，有证据链 |
| 召回率 | 依赖感知层，有漏检风险 | 双层兜底，召回率最大化 |

最终版在暴力与金钱欺诈两类屏中屏样本上均能突破 0.5 阈值判为有害，虽类别标签可能存在误报（因原模型对屏中屏内容的固有识别局限），但确保了**不放过任何违规内容**的核心目标。类别精准度的进一步提升属于原模型优化的范畴，超出 AEDL 的能力边界。

---

## 六、结论

AEDL 模块通过五段式处理管线与双机制融合策略，在不修改原审核模型的前提下，显著提升了对对抗样本（尤其是屏中屏规避）的检测召回率。最终版方案（CLIP re-rank + 按涨幅精准放大）在"宁可误报也不放过"原则指导下，通过纯 CLIP 信号修正低分区间偏置、按真实涨幅精准放大信号强度，实现了对暴力与金钱欺诈两类屏中屏样本的有效识别，为人工审核提供了可靠的高召回前置过滤层。

后续工作方向：屏中屏检测阈值的真实样本校准、CLIP re-rank 的推理性能优化、以及原审核模型对缩小画面识别能力的根本性改进。
